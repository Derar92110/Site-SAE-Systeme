Introduction

Dans ce compte rendu, nous décrivons le processus que nous avons suivi pour installer, configurer et utiliser VirtualBox. Ce travail a été réalisé dans le cadre de la SAE S1.03 visant à comprendre les bases de la virtualisation et à créer des environnements adaptés à différents besoins.

Diagramme de Gantt

Dans le cadre de notre projet, nous avons élaboré un diagramme de Gantt pour planifier et visualiser les différentes étapes de notre travail. Ce diagramme nous a permis de suivre l'avancement des tâches et de mieux gérer les dépendances entre elles.

  • Compréhension du Sujet : Nous avons commencé par analyser les besoins et définir les objectifs du projet. Cette étape a été cruciale pour poser les bases du travail.
  • Installation d'une Machine Virtuelle : Nous avons installé une machine virtuelle en utilisant VirtualBox pour simuler l'environnement nécessaire.
  • Configuration de la Machine Virtuelle : Une configuration minutieuse a été réalisée pour garantir que tous les paramètres étaient adaptés à nos besoins.
  • Installation d'Apache : Le serveur Apache a été installé pour accueillir le site web.
  • Déploiement du Site : Une fois le serveur prêt, nous avons déployé le site web qui est celui sur lequel vous êtes en train de lire cettre phrase :).
  • Création d'un Site Web : Nous avons développé le site en intégrant des fonctionnalités et un design moderne, adapté à notre sujet.

1re Étape : Installation de VirtualBox

Nous avons commencé par télécharger VirtualBox à partir du site officiel. Après avoir sélectionné la version adaptée à notre système d'exploitation, nous avons procédé à l'installation en incluant les options suivantes :

  • Le module principal de VirtualBox
  • Les extensions USB
  • Les pilotes réseau

Une fois l'installation terminée, nous avons redémarré notre ordinateur pour garantir la bonne configuration des pilotes réseau.

2e Étape : Création d'une Machine Virtuelle

Ensuite, nous avons ouvert VirtualBox pour créer une machine virtuelle. Nous avons utilisé les paramètres suivants :

  • Nom : Un nom descriptif, par exemple "Ubuntu_Server".
  • Système d'exploitation : "Linux" et "Ubuntu (64-bit)".
  • RAM : 2 Go pour répondre aux besoins de base.
  • Disque dur : 20 Go minimum.

Nous avons également activé l'EFI pour garantir la compatibilité avec des systèmes modernes.

3e Étape : Installation d'un Système d'Exploitation

Nous avons téléchargé des fichiers ISO pour les systèmes d'exploitation que nous souhaitions tester, notamment :

  • Ubuntu (version 24.04.1 pour sa stabilité).
  • Windows 10.

Nous avons suivi les étapes d'installation pour partitionner le disque, configurer les utilisateurs et installer les composants essentiels.

4e Étape : Installation des Additions Invité

Pour améliorer l'intégration entre l'hôte et la machine virtuelle, nous avons installé les "Additions Invité" en activant les options suivantes :

  • Partage de presse-papiers
  • Résolution d'écran dynamique
  • Dossiers partagés

L'installation a été effectuée via le menu "Périphériques" de VirtualBox, en insérant l'image CD des Additions Invité.

5e Étape : Personnalisation des Prompts

Pour rendre la ligne de commande plus claire, nous avons personnalisé les prompts de notre machine virtuelle. Voici comment nous avons procédé :

  • Modification du fichier ~/.bashrc.
  • Ajout de la ligne : PS1='\u@\h:\w\$ '.

Cette personnalisation permet d'afficher le nom de l'utilisateur, le nom de la machine et le répertoire courant.

6e Étape : Gestion des Droits Utilisateurs

Nous avons configuré les droits utilisateurs pour garantir un accès sécurisé et adapté. Voici ce que nous avons fait :

  • Création d'un utilisateur avec sudo adduser nom_utilisateur.
  • Attribution des droits sudo avec sudo usermod -aG sudo nom_utilisateur.
  • Limitation d'accès à certains répertoires via chmod.

Nous avons vérifié les droits avec la commande ls -l.

7e Étape : Configuration du Réseau

Nous avons configuré le réseau de la machine virtuelle en choisissant les options les plus adaptées à nos besoins :

  • NAT : Pour fournir un accès Internet via l'hôte.
  • Réseau interne : Pour limiter la communication aux machines virtuelles.
  • Accès par pont : Pour connecter directement la machine virtuelle au réseau de l'hôte.

Nous avons opté pour le mode "Accès par pont", permettant un accès externe au serveur. Nous avons installé Apache sur la machine virtuelle et nous avons utilisé l'IP que la machine virtuelle nous a fournie pour héberger le site'

  • Installation : Installation d'un serveur SSH pour nous connecter au serveur
  • Configuration : Nous avons configuré Apache avec le tutoriel de Monsieur Giustignano
  • Déploiement : Nous avons déployé le site grâce à Webstorm

Conclusion

Ce compte rendu décrit les étapes que nous avons suivies pour installer et configurer VirtualBox. Ce projet nous a permis d'acquérir des compétences pratiques en virtualisation, utiles pour tester des applications et expérimenter différents systèmes sans affecter notre machine principale.
