Définition de la Virtualisation

La virtualisation est une technologie qui permet de créer des ressources virtuelles à partir de ressources physiques. Ces ressources peuvent inclure des serveurs, des réseaux, des systèmes d'exploitation ou des applications. Elle permet d'exécuter plusieurs systèmes d'exploitation ou applications sur une seule machine physique, en optimisant ainsi l'utilisation des ressources et en réduisant les coûts matériels.

Différence entre Émulateur, Simulateur et Outil de Virtualisation

  • Émulateur : Reproduit le comportement d'un système ou d'un appareil différent. Par exemple, un émulateur permet d'exécuter des applications iOS sur Windows.
  • Simulateur : Crée un modèle approximatif d'un système pour tester des scénarios. Il est souvent utilisé en recherche ou pour des essais.
  • Outil de virtualisation : Fournit un environnement isolé et exécutable (comme une machine virtuelle) basé sur des ressources physiques.

Type de Virtualisation pour les Environnements VirtualBox, VMware Workstation et Stratus

  • VirtualBox : Virtualisation complète basée sur un hôte (host-based virtualization). Elle est utilisée pour des systèmes clients et des serveurs légers.
  • VMware Workstation : Conçu pour les environnements professionnels, il offre une virtualisation de type bare-metal et hôte. Il supporte les outils avancés comme la gestion de snapshots.
  • Stratus : Connu pour la tolérance aux pannes, il utilise des solutions de virtualisation orientées cloud avec des systèmes résilients.

Forme/Lieu de Sauvegarde de la Machine Virtuelle Créée

Forme : Les machines virtuelles sont enregistrées sous forme de fichiers d'image disque (VDI pour VirtualBox, VMDK pour VMware).

Lieu :

  • Local : Dans un répertoire spécifique sur le disque dur de l'hôte.
  • Cloud : Utilisation de services comme Google Drive ou AWS S3 pour sauvegarder les machines virtuelles.
  • Périphériques externes : Stockage sur des SSD ou disques durs externes pour des besoins de portabilité.

Explication des Choix Techniques et Justification

Choix Techniques Possibles

  • Choix de la RAM :
    • Minimum 2 Go pour un système Linux basique.
    • Allouer jusqu'à 8 Go pour des systèmes plus complexes comme Windows 10 avec plusieurs applications.
  • Taille du disque dur virtuel :
    • Systèmes légers : 20 Go.
    • Systèmes complexes : 50 Go ou plus, avec la possibilité d'extension dynamique.
  • Type de réseau :
    • NAT : Pour l'accès Internet de base.
    • Accès par pont : Pour simuler une intégration au réseau physique.
    • Réseau interne : Pour des tests en milieu isolé.

Justification des Choix

RAM et disque : Une configuration adaptée garantit des performances optimales tout en évitant un usage excessif des ressources.

Type de réseau : Le mode NAT est idéal pour une navigation simple, tandis que le mode à pont permet des tests réalistes dans des scénarios réels.

Sauvegarde : Utiliser un stockage cloud assure une protection contre les pannes physiques et facilite le partage entre équipes.
